About FTT Network

FTT is a Public Blockchain on block {{!getBlocknumber() ? 0 : getBlocknumber() | number}} connecting all major crypto currencies. We show the last 3 digits of the current block in the navigation bar.

FTT is based on the $, a stable and globally known store of value. This makes it easy to know what things cost and avoids volitility risk.
Add Bitcoin, Ether and other crytocurrencies to your FTT account. Use multiple cryptocurrencies on one network.

Think of FTT as your crypto checking account. It is to be used. Think of Bitcoin, Ethereum and other blockchains as savings accounts. They are for long term storage.
Our smart-contract based accounting system prevents fraud by allowing community to audit in real-time.

Imported cryto external addresses and amounts are tracked on a smart-contract. The smart contract also tracks the usage of crypto tokens on the FTT network.
We provide smart-contract tooling for engineers. These tools speed up development process with CI/CD integration and let developers focus on code instead of platform infrastructure.

Developers can immediately publish to our network with a hidden address to test or share with their users. If desired, once a smart contract passes a quality review we integrate it into our site for mass-adoption.

We also provide templates for standard smart-contracts so non-developers can quickly solve their business needs.
Users interact with FTT using a simple and easy to use browser interface. Connect quickly with phone or browser.
This makes it economically practical to develop micro services and services for 3rd world. Fees are set by smart contracts. This way pricing can be based on industry and use case.
Users demand fast response time. Theoretical support of 1,000 transactions per second (TPS). Can scale much higher as demand grows.
Smart contracts can execute external APIs. Send text, email, update database, trigger lambda, etc etc. We provide tools for smart contract events to call external APIs.
Smart contracts can be triggered by external APIs. We provide tools for engineers to call smart contract functions from external APIs.
We don't believe in white-papers and marketing fluff. We believe in code.

Roadmap

Our goal is to build a crypto platform that other engineers can build on top of. See FinTechToken GitHub for a detailed list of tasks and issues.
We believe blockchain can revolutionize organization structure. Imagine contributors working on projects without bueracratic hiearchy. Unlike open source, contributors get paid project tokens based on value of contributions. This will attract top talent.

To accomplish this we Token Sourced our code. This means the license to the code is owned by FTT token holders. Tokens go to FinTechToken GitHub contributors. Tokens also go to capital contributors.
Ultimately we want this roadmap to be prioritized by a community of capital contributors. We believe blockchain can revolutionize corporate structure by enabling the community to vote on product enhancements with tokens that go to engineers that implement those features.

However we believe many projects that attempt democracy too early end in unescapable conflict that destroys the project. Instead we will drive the project forward until the time when external contributions outweigh our own.
As we develop crypto functionality we also build out tooling to share with community.

Similar to App-Stores, developers can add "Smart Contract" applications to our network. These can be deployed as hidden addresses, or officially deployed after quality review.
