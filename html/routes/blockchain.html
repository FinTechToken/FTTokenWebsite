FinTechToken

FTT is a Public Blockchain connecting all major crypto currencies

FTT is on block {{!getBlocknumber() ? 0 : getBlocknumber() | number}}. We show the last 3 digits in the navigation bar at the top of your screen.

  1. Base currency is US Dollars
    Easy to know what things cost

  2. Major crypto currencies are tokenized
    Like a checking account, users can import/export to FTT only what is needed. Once imported users get cheaper and faster transactions than using the native BitCoin/Ether/etc blockchains.

  3. Accounting of tokinization is done via smart contract
    Community can verify digital assets that are imported/exported during tokenization process. This real-time smart-contract based accounting system prevents fraud.

  4. Anyone can publish solidity smart contracts
    Similar to an App-Store. Simple templates for non-developers to quickly use DLT for their business needs. Advanced tools for engineers. Speed up development process with CI/CD integration. Focus on code instead of platform infrastructure.

  5. No need for complex blockchain tools
    Users interact with FTT using phone or browser. Simple and easy to use interface.

  6. Gas that runs smart contracts is totally free
    This makes it economically practical to develop micro services and services for 3rd world. Fees are set by smart contracts. This way pricing can be based on industry and use case.

  7. Transaction happen approximately every second
    Users demand fast response time. Currently supports 1,000 transactions per second (TPS). Can scale much higher as demand grows.

  8. Trigger off-chain actions based on smart contract
    Smart contracts can trigger external APIs. Send text, email, update database, trigger lambda, etc etc.

  9. Execute smart contract based on real world events
    Smart contracts can be triggered by external APIs.

  10. FTT is a working product
    We don't believe in white-papers and marketing fluff. We believe in code.

Frequent Questions

What is CryptoPass?

CryptoPass provides access to all major blockchains & tokens in one place.


How does CryptoPass work?

CryptoPass is built on the same cryptography as the Ethereum network.

Your account/keys are stored in one smart contract. With that smart contract you can send transactions to any major blockchain.


Is CryptoPass secure?

Yes. All transactions are encrypted clientside with your account/key pair. Because encryption happens on the client, your secure data never travels unencrypted over the internet. Your secure data is always encrypted when stored.


What is the FinTechToken network?

The FinTechToken network tokenizes all major blockchains. You can use the network to transact with major cryptocurrencies with 0 fees at 1 second blocks.

You can also create your own smart contracts using any major cryptocurrencies.


Is FinTechToken verifiable?

Yes. We publish all blocks from the FinTechToken network on S3 every 10 minutes so anyone can verify transactions.

Our code is Token Sourced and available for anyone to read.


What is a pass phrase?

A pass phrase is a group of random words. The pass phrase unlocks your CryptoPass. Keep it secret and secure. If someone knows or guesses your pass phrase they can use your CryptoPass.


What if I forget my passphrase?

If a passphrase is lost, it is lost forever. The most secure thing is to protect your own passphrase.

We offer an optional service to store your passphrase in our digital vault. This way the pass phrase can be recovered. We also allow you to name a beneficiary.


Roadmap

High Level

Our goal is to build a central crypto platform that other engineers can build on top of. See FinTechToken GitHub for a detailed list of tasks and issues.

Our code is Token Sourced. This means the license to the code is owned by FTT token holders. Tokens go to FinTechToken GitHub contributors. Tokens also go to capital contributors.

Ultimately we want the roadmap to be prioritized by the community. However we believe many projects that attempt democracy too early end in unescapable conflict that destroys the project. Instead we will drive the project forward until the time when external contributions outweigh our own.

The first step is to build a platform that can perform basic crypto functions. As we develop those basic functions we build out tooling that we share with community.

Our vision is that Distributed Public chains is where long term value is held. But, private chains like FTT is where useful day to day things happen.

Similar to App-Stores we want to let developers add "Smart Contract" applications to our network. These can be deployed as hidden addresses, or officially deployed after our quality checks.

We also have the ability to spin up this blockchain and developer tooling for companies to build internal networks. We imagine FTT being a hub, with many private chain spokes built on our tech, or built on other tech. The key is we connect them to the outside world through our hub.
