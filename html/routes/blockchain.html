Roadmap

Our goal is to build a crypto platform that other engineers can build on top of.
We Token Sourced our code so FTT contributors can work on this project without bueracratic hiearchy. Tokens go to both GitHub project contributors and capital contributors.

Token Source is similar to open sourced projects with two main differences.
  1. Users of the code need a license (Revenue is collected via smart contract)
  2. Project contributors get tokens (Revenue is distributed to token holders via smart contract)
Long term the FTT roadmap will be prioritized by the community. The community can vote on product enhancements with tokens. Those tokens go to engineers that implement the feature.
Similar to App-Stores, developers can add "Smart Contract" applications to our network. These can be deployed as hidden addresses, or officially deployed after quality review.
Companies that want Private Blockchains can white label FTT tech. FTT is a hub, with many private chain spokes extending the tech into private chains.
